◆ setData()

bool MarkModel::setData ( const QModelIndex &  index,
const QVariant &  value,
int  role = Qt::EditRole 
)

Reimplemented from QAbstractItemModel::setData(const QModelIndex & index, const QVariant & value, int role)

Available role includes all enums in Qt::ItemDataRole.

There are some customized roles besides above. MetaDataRole means that this function call will set value as the meta data field of TreeItem*, which is the exact data stored in the TreeItem. TreeItemRole means that this function call will set value as the specific instance of TreeItem in the node represented by index. Under this situation, the type of internal data in value should be void* and the value of column() of index is ignored

Parameters
index
value
role
Returns
Returns true if this call is successful otherwise return false.
Note
If role is Qt::CheckStateRole, the function will be called recursively to apply the same setting for its children
Warning
When role is TreeItemRole, the old TreeItem* pointer would be deleted at once.
See also
TreeItem, Qt::ItemDataRole, MarkModelItemRole
