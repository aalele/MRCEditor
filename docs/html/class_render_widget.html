◆ ~RenderWidget()

RenderWidget::~RenderWidget ( )
override

Destroyes the render widget.

Note
disconnect the QOpenGLContext::aboutToBeDestroyed() signal from QOpenGLContext links to RenderWidget::cleanup() is necessary Because the context is not destroyed yet after the widget is destroyed. The signal still emits and will activate a empty slot and the program will crash.
See also
QOpenGLContext RenderWidget:cleanup()

Critical: The context will emit aboutToBeDestroyed() signal and will cause a crash even after this class was destroyed if below disconnection had not been called
