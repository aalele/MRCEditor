◆ takeMarkModel()

MarkModel * SliceEditorWidget::takeMarkModel ( MarkModel *  model,
bool *  success 
)
noexcept

Replaces the old mark model with the given model and returns the old one. A flag success refers to is set as true when replacement is successful if it's not nullptr, otherwise it's set as false.

A check would be applied for the given model in case that the mark model model matches data model. If it don't match or the data model is empty, a flag success refers to will be set as false and nullptr will be returned. Above two situations won't emit markModelChanged() signals. Others will emits markModelChanged() signal. model set as nullptr is equivalent to delete mark model and will emit markModelChanged() signal

Note
This function also emits markModelChanged() signal

◆ takeSliceModel()

AbstractSliceDataModel * SliceEditorWidget::takeSliceModel ( AbstractSliceDataModel *  model)

Replaces the old slice model with the given model and returns the old one.

The mark model corresponding to the slice model will be deleted once the old slice model is replaced, and a new mark model will be created. So it's caller's responsibility to save the corresponding mark model before the replacement takes place, and the old slice mark model and its ownership will be returned to the caller. Sets model as nullptr is equivalent to empty the slice data.

Note
This function also emits dataModelChanged() signal
